# 文件拷贝管理器 (Files Copy Manager)

一个功能强大的跨平台文件拷贝管理工具，基于 Flutter 开发。该工具提供了灵活的文件复制功能，支持多种高级特性，适用于各种文件管理场景。

## 功能特点

### 📋 基础功能
- **多配置管理**：支持创建和管理多个拷贝配置，方便不同项目或场景使用
- **源目录与目标目录选择**：直观的目录选择界面，支持浏览和打开目录
- **实时拷贝状态监控**：显示当前拷贝进度和状态信息

### 🔧 高级功能
- **排除路径管理**：
  - 支持添加特定文件或目录到排除列表
  - 提供专门的排除路径管理界面
  - 支持通配符模式匹配（如 `*.json`、`*/a/*.meta`）

- **自定义排除规则**：
  - 支持通配符模式：`*.json`（排除所有 JSON 文件）
  - 支持递归模式：`*/raw-assets/*.png`（排除 raw-assets 目录下的所有 PNG 文件）
  - 支持路径模式：`*/a/*.meta`（排除 a 目录下的所有 meta 文件）

- **选择性拷贝**：
  - 支持只拷贝特定文件或目录（非全部拷贝）
  - 可以精确控制哪些文件需要被拷贝

- **目标目录清理**：
  - 可选择在拷贝前删除目标目录中的所有文件
  - 避免旧文件残留问题

### 📊 日志与追踪
- **详细拷贝日志**：记录每次拷贝操作的详细信息
- **成功/失败状态追踪**：区分成功和失败的拷贝操作
- **文件大小统计**：显示每个文件的大小信息
- **错误信息记录**：当拷贝失败时记录具体错误原因

### 💾 数据持久化
- **配置保存**：所有配置自动保存到本地存储
- **导入/导出功能**：支持配置的导入和导出，便于备份和分享

## 安装与运行

### 系统要求
- Flutter SDK (版本 ^3.9.2)
- 支持 Windows、macOS、Linux、iOS 和 Android

### 安装步骤

1. 克隆或下载项目代码
2. 在项目根目录执行以下命令：

```bash
# 获取依赖包
flutter pub get

# 运行应用
flutter run

# 或构建桌面应用
flutter build windows  # Windows
flutter build macos    # macOS
flutter build linux    # Linux
```

## 使用方法

### 1. 创建拷贝配置
1. 打开应用后，在配置下拉菜单中点击"新增"按钮
2. 输入配置名称（如"项目备份"、"资源同步"等）
3. 点击"重命名"可以修改配置名称
4. 使用"删除"按钮可以移除不需要的配置

### 2. 设置源目录和目标目录
1. 点击"选择目录"按钮设置源目录（要拷贝的文件所在目录）
2. 点击"选择目录"按钮设置目标目录（文件要拷贝到的位置）
3. 可以使用"打开目录"按钮快速在文件管理器中打开对应目录

### 3. 配置拷贝选项
#### 选择性拷贝
- 勾选"拷贝所有文件和目录"：拷贝源目录下所有内容
- 不勾选时：手动添加需要拷贝的特定文件或目录
  - 点击"添加目录"选择要拷贝的子目录
  - 点击"添加文件"选择要拷贝的特定文件

#### 排除路径设置
1. 点击"添加目录"或"添加文件"将不需要的路径加入排除列表
2. 点击"自定义规则"添加通配符模式：
   - `*.json`：排除所有 JSON 文件
   - `*/a/*.meta`：排除 a 目录下的所有 meta 文件
   - `*/raw-assets/*.png`：排除 raw-assets 目录下的所有 PNG 文件
3. 点击"更好的查看"进入专门的排除路径管理界面

### 4. 执行拷贝操作
1. 根据需要选择"拷贝前删除目标目录下的所有文件"选项
2. 点击"开始拷贝"按钮启动拷贝过程
3. 观察拷贝状态信息了解当前进度

### 5. 查看拷贝结果
- 拷贝完成后可点击"查看拷贝日志"查看详细记录
- 日志包含时间戳、源路径、目标路径、文件大小和成功/失败状态

### 6. 配置管理
- **导入配置**：通过"导入"按钮从 JSON 文件导入配置
- **导出配置**：通过"导出"按钮将配置保存为 JSON 文件
- **管理配置**：通过"管理配置"按钮进入批量配置管理界面

## 应用场景

- **项目备份**：定期备份项目文件，排除临时文件和缓存
- **资源同步**：在开发过程中同步资源文件到不同目录
- **文件分发**：将特定文件集合分发到多个目标位置
- **版本管理**：管理不同版本的文件集合

## 技术架构

本应用采用 Flutter 框架开发，具有以下技术特点：

- **跨平台兼容**：一套代码支持多个平台
- **现代化UI**：使用 Material Design 设计语言
- **响应式状态管理**：使用 Stream 和 ValueNotifier 进行状态管理
- **异步文件操作**：安全高效的文件拷贝机制
- **数据持久化**：使用 shared_preferences 存储用户配置

## 依赖库

- `file_selector`：提供跨平台文件/目录选择功能
- `shared_preferences`：本地数据持久化存储
- `url_launcher`：用于打开文件夹在系统文件管理器中
- `path`：路径操作和规范化处理

## 故障排除

### 常见问题
1. **无法选择目录**：确保应用有访问文件系统的权限
2. **拷贝失败**：检查目标目录是否有写入权限
3. **配置丢失**：确认应用有读写本地存储的权限

### 性能优化
- 对于大量文件的拷贝，建议使用排除规则减少不必要的文件传输
- 使用选择性拷贝功能只拷贝需要的文件
